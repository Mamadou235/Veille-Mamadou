Qu’est-ce qu’une base de données?

Une base de données est une collection de données connexes qui représentent un aspect du monde réel.
Un système de base de données est conçu pour être construit et rempli de données pour une certaine tâche.

Qu'est-ce que le SGBD?

Database Management System (DBMS) en Anglais ou encore système de gestion de base de données (SGBD) en français est un logiciel permettant de stocker et de récupérer les données des utilisateurs tout en tenant compte des mesures de sécurité appropriées. Il se compose d'un groupe de programmes qui manipulent la base de données. Le SGBD accepte la demande de données d'une application et demande au système d'exploitation de fournir les données spécifiques. Dans les grands systèmes, un SGBD aide les utilisateurs et d'autres logiciels tiers à stocker et à récupérer des données.
Le SGBD permet aux utilisateurs de créer leurs propres bases de données selon leurs besoins. Le terme «SGBD» comprend l'utilisateur de la base de données et d'autres programmes d'application. Il fournit une interface entre les données et l'application logicielle.

Exemple de SGBD

Voyons un exemple simple de base de données universitaire. Cette base de données conserve des informations sur les étudiants, les cours et les notes dans un environnement universitaire. La base de données est organisée en cinq fichiers:
Le fichier ÉTUDIANT stocke les données de chaque élève
Les magasins de fichiers COURS contiennent des données sur chaque cours.
La SECTION stocke les informations sur les sections d'un cours particulier.
Le fichier GRADE stocke les notes que les élèves reçoivent dans les différentes sections
Le fichier TUTOR contient des informations sur chaque professeur.

Caractéristiques du système de gestion de base de données

  • Fournit la sécurité et supprime la redondance
  • Nature auto-descriptive d'un système de base de données
  • Isolation entre programmes et abstraction de données
  • Prise en charge de plusieurs vues des données
  • Partage de données et traitement des transactions multi-utilisateurs
  • Le SGBD permet aux entités et aux relations entre elles de former des tables.
  • Il suit le concept ACID (atomicité, cohérence, isolation et durabilité).
  • Le SGBD prend en charge l'environnement multi-utilisateurs qui permet aux
  • utilisateurs d'accéder et de manipuler des données en parallèle.
